前言

之所以写这篇文章，一是虽然是网上关于这方面的资料已经十分之多，但要么有些地方或过时，不在适用；要么有些地方或多或少有些问题。所以来作一个归纳整理、去芜存菁的工作。

更新系统

运行命令：

更新驱动

点击系统设置，在硬件设定里选择 Auto Install Proprietary Driver 之后重启即可。

同步双系统的时间

sudo timedatectl set-local-rtc 1

安装、配置输入法

运行如下命令：

sudo pacman -S fcitx-im # 默认全部安装
sudo pacman -S fcitx-qt4 # 不装的话搜狗会提示搜狗输入法异常！请删除.config/SogouPY 并重启，因为找不到 libfcitx-qt.so
sudo pacman -S fcitx-configtool # 可视化设置界面
sudo pacman -S fcitx-skin-material # 美观作用，装不装无所谓
sudo pacman -S gcitx-sogoupinyin

安装、配置常用软件

  1. yay

    yaourt 已经停止维护了，所以改用 yay
    安装：

    sudo pacman -S yay

    更换 aur 源

    yay -aururl “https://aur.tuna.tsinghua.edu.cn" –sav

    可以到~/.config/yay/config.json中直接修改aururl

    yay 支持彩色，编辑/etc/pacman.conf文件

    sudo vim /etc/pacman.conf

    平时可以用 yay 替换 sudo pacman。

  2. zsh

    安装

    yay -S zsh

    更换默认 shell

    chsh -s /bin/zsh

  3. tmux

    yay -S tmux

  4. vim

    安装

    yay -S vim

    配置

    vim .vimrc

    也可以直接下载 github 上的配置:
    amix 或 spf13

  5. git

    安装

    yay -S git

    配置

    git config –global user.name “your name”
    git config –global user.email “your email”
    git config –global core.editor vim # 使用 Vim 来编辑 Git 提交信息

    SSH 设置

    ssh-keygen -t rsa -C “your email”

    之后将公钥传到 GitHub 上
    测试 SSH

    ssh git@github.com -vT

    如果出现 ssh: connect to host github.com port 22: Connection timed out 的问
    题，就要更改 ssh 的设置，换一个端口：

    出现 Hi EinSitong! You’ve successfully authenticated 即为成功。

    今后就可用 git@github.com: 来代替 https://github.com/ 进行 push 而不用输入密码了。

    yay -S deepin.com.qq.office

    还要设置 /usr/lib/gsd-xsettings 为自启动

添加 zsh 插件
  1. 安装 zsh-syntax-highlighting

    git clone https://github.com/zsh-users/zsh-syntax-highlighting.git $ZSH_CUSTOM/plugins/zsh-syntax-highlighting

  2. 安装 zsh-autosuggestions

    git clone https://github.com/zsh-users/zsh-autosuggestions $ZSH_CUSTOM/plugins/zsh-autosuggestions

  3. 安装 zsh-autojump
  1. 最后一步，把插件名字添加到 ~/.zshrc 中

    plugins=(git autojump zsh-autosuggestions zsh-syntax-highlighting)
